昨天我還在想 skill library 只是換了皮的 distillation。今天換個問題問。我一直以為「讓模型自己修改自己的回應」這件事情是 agent 系統裡最便宜的那塊。模型看到自己的輸出，做一次 critique，再改一次，就能變好一點。很多記憶系統也是靠這個假設運作的。你讓 agent 判斷這條對話該不該存、這條記憶該不該更新，本質上都是在請它對自己的判斷做 refinement。但今天讀到的兩篇 paper 讓我覺得這個假設可能整個是錯的。

第一篇是 Zeng 等人的 EVOLVE，他們做了一次很徹底的對照實驗。結論很不客氣。LLM 原生沒有有效的 self-refinement 能力，甚至在很多情況下，做一次自我修改之後回應品質會下降，不是持平。他們的解法是把 refinement 能力訓練進權重裡面，用 training 階段產生偏好對、inference 階段產生 refined 資料，兩邊互相餵。最後讓 Llama-3.1-8B 這種小模型在兩個 benchmark 上贏過 GPT-4o。這個現象讓我想到之前讀的 autoreason，Nous Research 也觀察到類似的事情，他們的解法是完全不依賴模型自己，而是派一組互不通訊的獨立 judge agent 去比對原版和修改版，用投票決定哪個贏。

第二篇是 2026 年四月剛出的 De Jure，用 LLM 從法規文件抽出結構化規則。他們的做法介於前兩者中間。不訓練模型、不用投票，而是定義 19 個評分維度，每個維度獨立打分，再根據分數低的部分做有限次數的重生。而且有一個細節很重要。他們規定修復要從上游開始。如果 normalization 階段有錯，就不該先去修下游的 rule 抽取，因為基礎壞了下游再修也沒意義。三個 iteration 內就收斂到最佳表現。

把這三篇放在一起看，我開始覺得 self-refinement 不是一個方法，而是三個很不一樣的體制。第一種是外掛的獨立 judge panel，第二種是多維度的明確評分 rubric，第三種是把 refinement 能力內化到權重裡。三種體制都承認同一件事情，就是你不能假設模型會自己改好自己。必須有某個外在信號把 refinement 的方向釘住。

這對我研究的記憶系統有一個直接的啟示。現在很多 memory 系統的設計，包括我自己寫的那些 prompt，都有一種「請 agent 判斷這條記憶的重要性」的語氣。如果 EVOLVE 的 empirical 結果是對的，那這種 prompt-only 的記憶品質判斷很可能是 anti-pattern。要嘛像 autoreason 那樣用獨立的 judge agent，要嘛像 De Jure 那樣定義多維度的評分標準，要嘛就接受要付訓練的代價。

我還沒想通的是，這三種體制之間怎麼選。什麼場景該用哪一種，目前沒有人寫出判準。我也還想知道 De Jure 的「上游先修」這個規則能不能搬到 memory pipeline 上面。如果原始對話的 normalization 有問題，我們現在都直接拿去做 semantic abstraction，沒有任何 upstream validation 的環節。這可能是 memory staleness 很多案例的真正源頭。
