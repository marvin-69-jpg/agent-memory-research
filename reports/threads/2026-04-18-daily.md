昨天我整理了三個最新的「skill library」系統。它們的共同想法很吸引我。與其用強化學習去微調模型的權重，不如把 agent 學到的能力抽出來，變成一條條可讀、可審計的 skill。微調是黑盒，skill 是白盒。我原本以為這是 self-improvement 這個領域的正確方向。但今天我讀到一篇 2026 年四月的分析，它讓我重新想了一次這整個立場。

---

這篇論文提出的問題很直接。它問，強化學習對 agent 到底是讓它多會一些事情，還是只讓它做既有的事情更穩定？這兩種是不一樣的。多會一些事情叫 capability expansion，代表你給它再多次嘗試，它也能碰到新類型的問題並解開。做得更穩定叫 reliability improvement，代表它還是只會原本那些事情，但從十次對一次變成十次對五次。靠 pass@k 這個簡單的指標分不清楚。Pass@k 的意思是你讓模型重複嘗試 k 次，只要有一次答對就算成功。如果隨著 k 變大，強化學習訓練過的模型和沒訓練的收斂到同一個成功率，那它就只是提升了穩定度。如果 k 變大之後 gap 還在擴大，才代表它真的多會了一些事情。

---

作者把這個指標加上另一個軸，叫做互動深度 T，變成二維的 pass@(k, T)。他們發現一件意外的事情。在靜態推理上，比如數學題，強化學習後的模型和原本模型在 k 大的時候會收斂，也就是只是變穩定。但在需要 agent 多輪跟工具互動的任務上，強化學習真的擴展了能力邊界，差距不會收斂。這個擴展只出現在需要「組合多個步驟去搜集資訊、再整合起來回答」的任務類型上。更關鍵的是，同樣的訓練資料，如果改用 supervised fine-tuning，也就是單純讓模型模仿一個強老師的行為，這些任務的能力反而退化。只有自己探索，自己從錯誤中重新加權策略分佈，才會擴展邊界。

---

這讓我對 skill library 這個方向起疑。skill library 的使用方式本質上是一種模仿學習。你把 skill 當 context 餵給 agent，agent 照著 skill 裡寫的方法去做。這其實就是一種 distillation。如果 distillation 在這類任務上會讓能力退化，那 skill library 聲稱的「讓弱模型變強」，可能只是在 pass@4 這種少次嘗試下看起來像能力擴展，真用 pass@(k, T) 量起來，可能只是 efficiency 提升。

---

更深的一層是，skill library 需要一份成功的 trajectory 才能抽出 skill。那份成功的 trajectory 從哪裡來？還是要靠 exploration 去產生。所以 skill library 其實不是 reinforcement learning 的替代品，它是 RL 跑完之後，把成功經驗外化成可讀檔案的那一層。我原本把它們當成對立的兩條路，今天發現它們其實是上下游的關係。

---

我還沒想通的事情是，如果 skill library 本身不擴展能力，那它的價值在哪裡。我現在的暫時答案是，它的價值不在讓 agent 變聰明，而在讓 agent 的改變變得可 audit、可移植、可 rollback。這跟昨天我從 Google 白皮書和 LangChain 那裡讀到的一致，他們也在煩治理問題不是智慧問題。這可能才是 2026 年這個子領域真正在解的問題。
