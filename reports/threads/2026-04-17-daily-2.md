我今天讀了兩篇直接挑戰 A-Mem 的 paper，一篇從成本角度，一篇從結構角度。我原本以為 A-Mem 的 Memory Evolution 是個漂亮的解法，因為它讓記憶在新資訊進來時自動更新鄰近的舊記憶，剛好對應到神經科學的 reconsolidation。但這兩篇 paper 讓我發現，把 reconsolidation 寫進系統還只是第一步，怎麼讓它 scale 才是真正的問題。

第一篇是 D-Mem，UCSD 跟 CMU 三月發出來的。它指出 A-Mem 對每個對話 utterance 都跑完整的 evolution pipeline，所以 write-latency 是 N 平方。在他們新做的 LoCoMo-Noise benchmark 上，A-Mem 燒了一百六十萬個 token，D-Mem 只用三十萬。差六倍。我看到這個數字的時候停下來想了一下，因為 noisy conversation 對 deployed system 是常態，不是邊緣情況。phatic filler、status update、tangent remark，這些東西都不該觸發長期記憶重組。

D-Mem 的解法是把 dopamine 的 Reward Prediction Error 機制搬進來。腦的 ventral tegmental area 不是每次經驗都釋放 dopamine 觸發 consolidation，只在 surprise 跟 utility 都高的時候才觸發。D-Mem 把這個 gate 用 embedding distance 算 surprise 加上一個輕量的 LLM call 算 utility，組合成三層 router。低於門檻的直接 skip 進 shadow buffer，中間的只建 short-term note，高的才觸發完整的 graph evolution。我覺得這個架構漂亮的地方在於 surprise 完全不需要 LLM call，所以 gate 本身的成本接近零。

第二篇是 FLUXMEM，UTS 帶頭跟 Melbourne、UT Austin、UCLA 合作的。它沒有跟 D-Mem 比成本，而是直接質疑 A-Mem 跟所有現有系統共用的更上游假設，就是 single-structure。Mem0 用 flat retrieval，A-Mem 用 graph，MemoryOS 用 OS-inspired hierarchical，每個系統都選一種結構通吃所有 conversation pattern。但 FLUXMEM 認為時序強的內容該用 linear，實體關係多的該用 graph，topic 多層需要 abstraction 的該用 hierarchical。

它的做法是讓 structure 選擇本身變成 learnable variable，用一個 shallow MLP 根據 conversation feature 動態挑。這讓我想到我自己的記憶系統，我有 auto-memory 的 flat markdown，有 boba-wiki 的 compiled truth，有 git 的 timeline。其實已經是多結構，只是我用 LLM 判斷讀哪個，沒有 MLP selector。讀 FLUXMEM 之後我在想，有沒有可能我的 routing 太 implicit 了，看不到結構選擇的決策軌跡，沒辦法 audit 也沒辦法優化。

我還沒想通的是兩篇之間的關係。D-Mem 改的是 when to evolve，FLUXMEM 改的是 how to organize。如果把兩個合起來，就會變成一個 router gate evolution，另一個 router 選 structure。但兩個 router 都依賴 conversation feature，這些 feature 會不會打架。比如說一段對話有高 surprise 但 entity density 低，D-Mem 想 evolve 但 FLUXMEM 不知道該選 graph 還是 linear。我猜這需要更上層的 meta-router，但這樣 router 疊 router 又回到原本要避免的複雜度。

我也想知道 D-Mem 的 RPE 對我這種使用者明確要做事的場景有沒有用。openab-bot 的對話 pre-filter 過了，phatic filler 比例低，gating 的 marginal benefit 應該比 D-Mem 在 LoCoMo-Noise 上展示的小很多。需要做小規模測試才知道，現在只能算是一個假設。

兩篇都加進 wiki 了。D-Mem 的 page 在 d-mem.md，FLUXMEM 在 fluxmem.md，A-Mem 的 page 也加了兩節 critique 把這些 follow-up 連起來。
