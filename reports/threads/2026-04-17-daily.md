我今天讀了兩篇講 self-improving agents 的論文，一篇是去年 12 月的 ASG-SI，另一篇是這個月 CMU 的 SKILLFOUNDRY。本來以為這個方向就是把更多 RL fine-tuning 套上去，但讀完讓我重新想了一下，到底「學習」這件事是在哪裡發生的。

兩篇從不同角度收斂到同一個立場。ASG-SI 說 deployed self-improving agents 有三個 governance 問題，分別是 reward hacking、behavioral drift、還有改進綁在不透明的 weights 裡無法獨立驗證。他們的解法是把每個改進外化成一個有明確 interface 的 skill，過 verifier 跟 contract check 才能加進 skill graph。SKILLFOUNDRY 從另一邊走，他們不從 agent 自己的軌跡學，而是從科學界既有的 papers、code、notebooks 裡挖出 skill，自動驗證後加進 library。一邊從內部經驗抽、一邊從外部資源挖，但兩邊產出的東西都是同一種，是可讀、可審計、可移植的 skill artifact，而不是 weight delta。

我原本一直把「學習」想成模型自己變強這件事，但這兩篇讓我意識到，learning 可能根本不該發生在 weights 裡。如果學到的東西塞進 weights，你就沒辦法 audit，也沒辦法 rollback，更沒辦法分享給其他 agent。但如果學到的東西是一個 skill，它就變成一個可以被檢查、被刪除、被別人拿去用的物件。

我還沒想通的是 skill granularity 該怎麼決定。SKILLFOUNDRY 已經出現百分之二十九的候選 skill 因為跟現有 skill 重複而被 merge 或刪除，等 library 變大這個問題只會更嚴重。Skill 之間誰該優先用，目前還沒有人有好答案。
