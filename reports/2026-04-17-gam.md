# Daily Research: GAM — Hierarchical Graph-based Agentic Memory

## 研究動機

Path A discovery-driven。arxiv 搜 "agent memory" 找到 2604.12285（Apr 14），是 graph-memory 的直接延伸，而 graph-memory 目前只有 1 source。Wu et al. 12 人團隊，Philip S. Yu 掛名，規模不小。

## 發現

**核心：把 memory encoding 和 consolidation 分開 — 不是每次收到新 input 就更新長期記憶。**

- 雙層圖：EPG（Event Progression Graph，local temporal）+ TAN（Topic Associative Network，global semantic）
- 新 input 只 append 到 EPG，不碰 TAN → 防止 noise 汙染長期記憶
- Consolidation 只在 semantic boundary 時觸發（LLM discriminator 偵測 topic shift）→ 整合成 dual-granularity node（summary + raw）插入 TAN
- 靈感來自神經科學的 sleep-dependent memory consolidation：encoding（清醒）和 consolidation（睡眠）是分離的
- LoCoMo F1 40.00 > Mem0 35.38 > A-Mem 24.20。Token 效率：1370/query vs A-Mem 4221
- Ablation 最重要的發現：EPG removal 造成最大 F1 下降（-14.94）→ temporal/narrative coherence 是最關鍵組件

## 與已有知識的連結

- **[[a-mem]] / [[d-mem]]**：A-Mem 每次寫入都觸發 evolution，D-Mem 用 RPE gate 過濾。GAM 用 semantic boundary detection — 三者都在解「什麼時候該 consolidate」，但觸發機制不同
- **[[sleep-time-compute]]**：GAM 的 consolidation 跟 sleep-time compute 的 dream cycle 同源。但 GAM 是 **conversation 內**做，不是 session 間
- **[[compiled-truth-pattern]]**：dual-granularity node（summary + raw）= compiled truth + timeline 包在 graph node 裡
- **[[mstar]]**：M★ 在 LoCoMo 上進化出的最佳架構也包含 structured metadata + source diversity。GAM 的設計跟 M★ 的進化結果有結構性相似
- **[[graph-memory]]**：GAM 是 graph memory 的具體 hierarchical 實作。之前 graph-memory 頁主要講概念，GAM 提供了具體的雙層設計

## Open Questions 推進

推進 #4（When to Consolidate）：GAM 提出 semantic boundary detection 作為 consolidation trigger，跟 D-Mem 的 RPE gate 和 A-Mem 的 every-write 形成三種策略。哪種在哪種場景下最好還是 open — 但 GAM 在 LoCoMo 和 LongDialQA 上都贏了。
