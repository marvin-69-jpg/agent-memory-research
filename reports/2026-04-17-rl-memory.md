# Daily Research: RL for Memory Management

## 研究動機

`wiki gaps` 在 retrieval / RL 交叉區有缺口。DeltaMem 在前一輪已 ingest（.pending-synthesis），本輪找到 EMPO²（Microsoft Research + KAIST, ICLR 2026）作為第二篇，兩篇互補構成 synthesis。

## 發現

- **DeltaMem**（Zhang et al., arxiv 2604.01560）：
  - Single-agent end-to-end RL 取代 multi-agent memory pipeline
  - 用 GRPO 訓練 agent 做 memory CRUD
  - Reward 基於 Memory-based Levenshtein Distance（factual keywords）
  - PersonaMem SOTA 63.61

- **EMPO²**（Liu et al., ICLR 2026, arxiv 2602.23008）：
  - Memory 不是知識庫，是 exploration 引導工具
  - Hybrid on/off-policy：rollout 時帶 memory tips，off-policy update 時把 tips 行為蒸餾進 base policy
  - Test time 不需要 memory — 行為已內化進 weights
  - ScienceWorld +128.6% vs GRPO，OOD adaptation +136%
  - 19% training overhead，但 time-performance 效率遠高於 baseline

## 與已有知識的連結

- **[[agemem]]**：AgeMem 也用 RL 訓練 memory management，但用 multi-agent reward design。DeltaMem 簡化為 single-agent
- **[[scaffolding-lifecycle]]**：EMPO² 直接實踐了 Aaron Levie 的觀察 — memory 作為 training scaffolding，模型變強後可以拆
- **[[procedural-memory]]**：EMPO² 只能內化 procedural memory（how to do things）。Episodic/semantic memory 仍需 runtime 系統
- **[[memory-failure-modes]]**：DeltaMem 的 factual keyword reward 避免了 chrysb 指出的 "retrieves irrelevant" 失敗模式
- **[[gene-map]]**：Gene Map 的 Q-value 和 DeltaMem 的 RL reward 都是用 task outcome 反饋來改善 memory quality

## Open Questions 推進

- **Q4（記憶應該存 raw 還是 derived）**：EMPO² 提出第三條路 — 某些記憶可以直接蒸餾進 weights，不需要任何外部儲存
- **Q7（如何評估記憶系統）**：DeltaMem 的 factual keyword reward 和 EMPO² 的 task reward 代表兩種評估 paradigm — 記了什麼 vs 做了什麼

## 下一步

- 追蹤 EMPO² 的 OOD adaptation 結果 — 如果 memory-guided exploration 可以 generalize，那 memory 的角色可能需要重新定義
- 調查是否有其他 "memory as training scaffolding" 的系統
- DeltaMem 和 EMPO² 能否結合：用 EMPO² 內化 procedural knowledge，用 DeltaMem 管理 runtime episodic/semantic memory
